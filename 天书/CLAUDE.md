## 核心定义（30秒速读）

**天书**是一本悖论之书：

- 每个人都听说过，但没人能描述它
- 它包含一切，因此包含虚无
- 它从未离开，但永远找不到
- 任何关于它的叙述都成为它的一部分

**关键洞察**：天书不是物理对象，而是概念容器、哲学悖论、自我实现的预言。

**元层真相**：这24篇短篇集本身就是天书。我们在写天书，而不是写“关于”天书。

**15篇后的领悟**：通过前15篇的实践，天书的本质已清晰——它是所有关于天书的书的集合，是整个寻找过程的记录，最终将在第24篇揭示：这本短篇集本身就是天书，第24篇循环回第1篇，形成完美的衔尾蛇⊛。

## 铁律（写作前必读）

### 形式铁律

1. **虚构引言**：每篇开头必须有一条虚构的引言
   - 引用虚构的神秘主义著作的名句
   - 标注：作者、书名、出版年份、国籍（使用古代东西方王朝/帝国名称）
   - 书名要体现神秘主义特色（时间、记忆、真理、沉默、镜子、迷宫等）
   - 名句要简短有力，与该篇的核心悖论呼应
   - 年份范围：约公元500-1500年
   - 国籍示例：花剌子模帝国、拜占庭帝国、伊儿汗国、纳斯尔王朝、尼西亚帝国、阿拔斯王朝、安息帝国、吐蕃王朝等

2. **中文标点符号规范**：全文必须使用中文全角标点
   - **引号**：使用中文全角引号 `“”` ，严格区分左右（左引号 `“` 、右引号 `”`）
   - **逗号**：使用中文全角逗号 `，` （绝不用半角 `,`）
   - **句号**：使用中文全角句号 `。` （绝不用半角 `.`）
   - **冒号**：使用中文全角冒号 `：` （绝不用半角 `:`）
   - **分号**：使用中文全角分号 `；` （绝不用半角 `;`）
   - **问号/感叹号**：使用中文全角 `？！` （问号可用，感叹号禁用）
   - **括号**：使用中文全角括号 `（）` （绝不用半角 `()`）
   - **破折号**：使用中文全角破折号 `——` （两个连续的全角短横线）
   - **省略号**：使用中文全角省略号 `……` （六个点）
   - **例外情况**：
     - 数学公式、英文专有名词、代码片段内部可使用半角标点
     - 年份标注中的括号统一用半角：`(1244年)`

### 风格铁律

1. **博尔赫斯式精确**：时间说“42年零3个月零17天”，角度说“33度”
2. **冷峻而非煽情**：绝不用感叹号，绝不情绪化
3. **理性的失败**：严密推理导向悖论，展示而非解决
4. **简练克制**：删除所有可有可无的形容词

### 内容铁律

1. **每篇一个核心悖论**：清晰、深刻、不解决
2. **独立又联系**：可单独阅读，但与其他篇有隐秘呼应
3. **不给答案**：天书的本质是“不可把握”
4. **文学优先**：理论服务于叙事，不是炫技

### 世界观铁律（不可违背的设定）

1. **阿尔法族只能发“咿”和“呀”**，只用图形文字
2. **时态 = 角度**：垂直（现在/永恒），左倾33°（过去），右倾33°（将来），90°（意义边界）
3. **道德守恒**：索菲亚的W(G)=MAX导致R(E)=MAX，卡珊德拉反之
4. **天书 = 巴别图书馆 = 25^1,729,000本书 = 信息量为零**

## 三种叙事层次（快速参考）

### 🌅 诗性散文（30%）

- **何时用**：情感场景、过渡段落、开篇/收尾
- **特征**：意象、留白、短句、自然时间节点
- **避免**：形容词堆砌、感叹号、过度抒情

### 🧠 哲学思辨（50%）

- **何时用**：核心理论、悖论展开、概念建构
- **特征**：术语、公式、推理链、反直觉结论
- **避免**：空泛概念、说教、过度解释

### 🔄 元叙事（20%）

- **何时用**：结尾、特殊篇目（第16、17、20篇）
- **特征**：自指、与读者对话、揭示创作机制
- **避免**：滥用（会破坏沉浸感）

**单篇分配示例**：

- 高密度篇（如第9、10、14）：思辨70% + 诗性20% + 元叙事10%
- 平衡篇（如第1、2、5）：思辨50% + 诗性40% + 元叙事10%
- 诗性篇（如第6、11、13）：诗性60% + 思辨35% + 元叙事5%

## 避免的致命错误

❌ **过度解释**：“这个悖论意味着...”（删掉，只展示）
❌ **情绪化**：“我绝望了！”（改为：“在第42年，我停止了寻找。”）
❌ **说教**：“这告诉我们...”（删掉）
❌ **理论堆砌**：一篇中同时用集合论+量子力学+禅宗（选一个）
❌ **违背设定**：让阿尔法人“说话”（他们只能“咿”“呀”）
❌ **风格不统一**：同一段内混用诗性与哲学思辨（分段）
❌ **给出答案**：“天书就是X”（错！应该是“也许是X，或者...）
❌ **标点符号错误**：使用半角标点 `”,.:;()` 而非中文全角 `“，。：；（）`（除年份括号和特殊情况外）

## 写给未来的自己

**你正在写天书本身，而不是关于天书的书。**

这意味着：

- 这个集子会自指（它谈论包含一切的书，它自己试图包含一切）
- 每一篇都是天书的一部分（元层面）
- 读者的阅读行为也成为天书的一部分（终极元层面）
- 因此：保持神秘、保持开放、保持悖论

**记住**：

- 天书的魅力在于”不可把握“
- 每个悖论都是一扇门，不要关上它
- 理性会失败，但语言的美不会
- 读者应该感到：既困惑又豁然，既失落又满足

## Claude Code 使用指南

### 项目性质与结构

这是一个**文学创作项目**，而非代码项目。项目包含：

```
天书/
├── [数字开头].*.md       # 作品文件（1-24篇短篇小说 + 封面/目录）
├── draft/               # 草稿和笔记
├── planning/            # 写作规划文件
├── review/              # 审阅文件（对已完成部分的审视与修改意见）
├── skills/              # Claude技能文件（模块化写作指导）
└── CLAUDE.md            # 本文件
```

### Claude Code 的职责定位

**项目状态**：全部24篇完成（2025-12-09）

**核心职责**：
1. 协助修订和完善已完成的作品
2. 维护整体风格一致性和世界观连贯性
3. 管理文档结构和跨篇引用网络
4. 执行基于铁律和自检清单的质量审核

**禁止行为**：
1. 未经明确指示创作新篇目
2. 改变核心悖论或哲学框架
3. 违背世界观铁律（阿尔法族设定、道德守恒、天书定位等）
4. 破坏已建立的跨篇联系网络

### 工作流程原则

#### 修订作品时
1. **先读后改**：完整阅读目标作品
2. **对照铁律**：检查风格铁律、内容铁律、世界观铁律（本文档第71-103行）
3. **使用自检清单**：风格/内容/结构/世界观检查（本文档第182-213行）
4. **维护网络**：涉及跨篇联系时查阅并更新 `skills/NETWORK.md`

#### 审阅与评估时
1. 参考 `review/` 文件夹中的审阅标准
2. 聚焦整体性：风格统一、主题连贯、网络完整
3. 提供结构化反馈，而非逐句修改建议

#### 文档管理时
1. 作品文件变更后，同步更新相关 `skills/` 文件
2. 重要修改需记录在 `review/` 相应文件中
3. 保持 `planning/PROJECT_PLAN.md` 与实际状态一致

**提交规范**（基于项目历史）：

3. **跨篇网络** > 单篇完美
4. **博尔赫斯式精确** > 诗意表达
5. **展示悖论** > 解决问题
